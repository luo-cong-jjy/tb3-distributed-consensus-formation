---

```

# 开启详细日志（查看每次合并的详细信息）
roslaunch tb3_distributed_control test_clustering_analysis.launch robot_id:=4 verbose:=true
```

## ⚙️ 参数调整

如果需要调整合并行为，修改 `test_clustering_analysis.py` 第298-300行：

```python
merge_spatial_threshold = 0.25  # 增大 → 更激进合并；减小 → 更保守
merge_angle_threshold = 0.3     # 增大 → 允许更大角度跨度
blind_zone_threshold = 0.30     # 增大 → 更宽松的盲区判断
```

**调整建议**：
- 如果合并太频繁（误合并）→ 减小 `merge_spatial_threshold` 到 0.20
- 如果合并太少（漏掉盲区）→ 增大 `blind_zone_threshold` 到 0.35
- 如果角度判断太严格 → 增大 `merge_angle_threshold` 到 0.4

---

## 📈 性能指标

**正常性能范围**：
- `merge_close_adjacent` 耗时：0.5-3.0 ms
- 合并率：0-15%（取决于环境和机器人行为）
- CPU利用率增加：< 3%

**如果超出范围**：
- 耗时 > 5ms → 检查是否有过多的聚类需要处理
- 合并率 > 30% → 阈值设置可能过于激进

---

## ⚠️ 注意事项

1. **verbose模式会影响性能**
   - 建议正常测试时使用 `verbose:=false`
   - 只在需要详细调试时开启

2. **测试环境要求**
   - 需要有墙壁或障碍物
   - 机器人需要能够接近到 < 0.3m 距离

3. **安全距离**
   - 手动控制时注意不要撞墙
   - 测试时速度不要太快

## 🐛 故障排查

### 问题1：从未发生合并
**可能原因**：
- 机器人距离障碍物始终 > 0.3m
- 阈值设置过于严格

**解决方案**：
```bash
# 手动控制机器人更靠近墙壁
# 或调整 blind_zone_threshold 到 0.40
```

### 问题2：合并过于频繁
**可能原因**：
- 阈值设置过于宽松
- 将不同障碍物误合并

**解决方案**：
```python
# 减小空间距离阈值
merge_spatial_threshold = 0.20  # 从 0.25 减小到 0.20
```

### 问题3：性能下降明显
**可能原因**：
- verbose模式开启
- 分区数量过多

**解决方案**：
```bash
# 关闭verbose模式
roslaunch tb3_distributed_control test_clustering_analysis.launch verbose:=false
```
